# Coding Standards

- Write the simplest code that solves the problem. Optimize only when there is a measured need.
- Handle errors explicitly. Never silently swallow exceptions.
- Avoid deep nesting — prefer early returns and guard clauses.
- Name things for what they are, not what they do internally.

## Principles

- Clarity over cleverness.
- Explicit over implicit behavior.
- Small, focused units of code.
- Maintainability over short-term speed.

## Expectations

- Code should be easy to read and understand without external explanation.
- Logic should be straightforward and predictable.
- Side effects must be intentional and visible.
- Edge cases should be handled deliberately, not accidentally.
